# 默认动作库配置表

本文件为桌面萌宠工坊的标准动作库参考，步骤 4 动效生成与步骤 6 说明交付时均参考此格式输出。

## 默认 5 套基础动作

| 动作名称 | 动作效果 | 适用场景 | 触发方式 | 时长建议 | 循环类型 |
|-----------|----------|----------|----------|----------|----------|
| 待机呼吸 | 身体轻微起伏、缓慢眨眼的闲置状态 | 无操作时默认播放 | 自动（默认状态） | 3-4 秒 | 无缝循环 |
| 开心蹦跳 | 上下跳跃、小幅摆尾/摆手的活泼动作 | 点击互动触发 | 鼠标点击萌宠 | 2-3 秒 | 单次播放后回归待机 |
| 软萌撒娇 | 歪头、蹭动的亲昵动作 | 鼠标悬停触发 | 鼠标悬停在萌宠上 | 3-4 秒 | 悬停期间循环，移开回归待机 |
| 犯困打盹 | 缓慢眨眼、低头耷拉的困倦状态 | 长时间无交互触发 | 连续 30 秒无交互自动切换 | 3-4 秒 | 无缝循环 |
| 挥手打招呼 | 抬手/抬头的欢迎动作 | 唤醒/手动切换 | 手动选择或唤醒触发 | 2-3 秒 | 单次播放后回归待机 |

## 自动触发逻辑规则

| 条件 | 触发动作 | 后续行为 |
|------|----------|----------|
| 无任何操作 | 待机呼吸 | 持续循环播放 |
| 连续 30 秒无交互 | 犯困打盹 | 持续循环，直到新交互发生 |
| 鼠标点击萌宠 | 开心蹦跳 | 播放 1 遍后回归待机呼吸 |
| 鼠标悬停在萌宠上 | 软萌撒娇 | 悬停期间循环播放，移开后回归待机 |
| 手动切换/唤醒 | 挥手打招呼 | 播放 1 遍后回归待机呼吸 |

## 自由切换逻辑

- 支持用户手动选择任意动作播放，无强制触发顺序
- 切换动作时过渡自然，无画面断层
- 用户自定义动作可直接追加到此动作库

## 动效生成提示词模板

为每个动作生成 ImageGen 描述时，可参考以下模板结构：

```
Q版萌宠 [角色描述]，[动作描述]，透明底，软萌质感，平涂风格，
[特征保留关键词]，循环动画关键帧，2-4秒时长，
无缝循环，无背景，PNG透明通道
```

**示例 — 待机呼吸动作：**
```
Q版萌宠小猫，身体轻微起伏呼吸，缓慢眨眼，
透明底，软萌质感，平涂风格，保留橘色毛色特征，
循环动画关键帧，3秒时长，无缝循环，无背景，PNG透明通道
```

**示例 — 开心蹦跳动作：**
```
Q版萌宠小猫，上下跳跃蹦跳，小幅摆尾巴，
透明底，软萌质感，平涂风格，保留橘色毛色特征，
循环动画关键帧，2秒时长，单次播放，无背景，PNG透明通道
```

## 交付文档输出模板

成品交付时，按以下格式输出动作清单与交互说明：

```markdown
# 桌面萌宠 — [角色名称] 动作清单与交互说明

## 动作清单

| 序号 | 动作名称 | 动作效果 | 触发方式 | 时长 | 循环方式 |
|------|----------|----------|----------|------|----------|
| 1    | 待机呼吸 | ...      | ...      | ...  | ...      |
| 2    | 开心蹦跳 | ...      | ...      | ...  | ...      |
| ...  | ...      | ...      | ...      | ...  | ...      |

## 交互规则

- 默认状态：待机呼吸（无缝循环）
- 点击触发：开心蹦跳（单次播放 → 回归待机）
- 悬停触发：软萌撒娇（悬停循环 → 移开回归待机）
- 闲置触发：犯困打盹（30秒无交互 → 循环播放）
- 手动切换：支持任意动作自由切换

## 文件清单

| 文件名 | 动作对应 | 格式 | 尺寸 |
|--------|----------|------|------|
| [角色名]_idle.gif | 待机呼吸 | GIF | 128×128 |
| [角色名]_happy.gif | 开心蹦跳 | GIF | 128×128 |
| ... | ... | ... | ... |

## 桌面使用小贴士

- 推荐尺寸：128×128 或 256×256 像素
- 格式兼容：透明底 GIF / PNG序列帧 / APNG
- 常见桌面宠物框架兼容建议：Shimeji、Desktop Goose 等
```

## 自定义动作扩展说明

用户可按需补充自定义动作，新增动作需提供以下信息：

- 动作名称
- 动作效果描述
- 适用场景
- 触发方式（自动/手动/交互触发）
- 时长建议（2-4秒）
- 循环类型（无缝循环/单次播放）

收到自定义需求后，按默认动作库格式追加到动作清单中，并同步生成对应动效素材与交互规则。
